# 智慧医疗前端 UI 设计与 API 交互准则

## 一、 核心设计理念 (KISS 极简原则)
1. **大气与严肃性兼具**：医院管理级平台，杜绝花里胡哨和娱乐化的过度渲染。UI 色调应沉稳（深蓝、浅灰、纯白），配以毛玻璃质感增强空间感，但不可喧宾夺主。
2. **第一性原理诊断规范**：前后端联调不仅浮于表面，需探究根本。图表不显示？先查 API。API 404？查 Controller 路径与配置。坚决摒弃靠凭空“试试看”去解决排查问题。

## 二、 交互与响应式准则
1. **无死角联动**：科室驾驶舱和聚合门户等复杂 Dashboard 区域，必须保证“牵一发而动全身”。切换主维度时（如下拉选择科室），需同时更新数字看板、列表排名及核心 Echarts 视图的入场动画。
2. **报错降级体验**：
   - 绝不允许“白板”、“空白 Drawer”现象，需提供“暂无数据”、“服务异常”等兜底文案。
   - 所有长列表必须有适当高度与滚动条或分页控制。
3. **针对 Agent 的容错指引**：
   - **自动化加载避坑**：在联调阶段，当前端服务启动在 Vite (5173) 时，由于 HMR 长连接的干扰，自动化探查工具（如 Playwright 或 Agent）的 `NetworkIdle` 事件绝不要设置为唯一依据，必须加入超时捕获或改为 `DOMContentLoaded` 断言。
   - **优先级指向**：尽可能去测 8080 JAR 包生产集成版。
