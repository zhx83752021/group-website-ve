# 智慧管理平台 - 全站深度回归测试工作流 (Workflow)

本文档旨在针对近期（包括今日）的所有修复、重构及接口连通性调整，提供一份**端到端的前后端系统验收级测试流程指南**。测试维度涵盖：
- **后端**：JUnit 单元测试（SpEL 算子引擎、公式校验、同环比分析）
- **前端 E2E**：Playwright 脚本自动化（登录、指标库交互、驾驶舱联动、门户检索）
- **手动/Agent 交互验证**：角色权限隔离、审计日志、字典联动等无法完全脚本化的深度链路

请按照下方的主章节逐一执行。

## ⚠️ 零、 AI Agent 执行规范与异常处理 (Hooks & Fallbacks)

1. **入口环境选择**：请优先测试 **http://localhost:8080**（生产打包模拟）。**严禁毫无防备地使用 `localhost:5173` 进行 NetworkIdle 级别网络空闲等待**，因为 Vite HMR 的热更新 WebSocket 会导致页面加载事件永久挂起（假死）。
2. **账号凭据脱敏**：执行登录时，不要在日志中输出硬编码明文密码。请提取系统环境变量中的 `TEST_ADMIN_PWD` 密码，若无，则默认尝试通用测试密码（见本地 sql 脱敏说明），请遵循最新的安全隔离最佳实践。
3. **异常自反馈流 (Self-Feedback Loop)**：
   - 每执行完一个链路，若遇到长时间无响应（>10s），必须立即停止当前执行并调用 `browser_console_check` 获取错误堆栈。
   - 若发现图表无数据或产生白屏，指导自己自动调用 `webapp-testing` 技能捕捉界面截图，然后使用 `frontend-design` 审查 ECharts DOM 结构或 API 拦截器源码。
4. **截图存证**：每个链路完成后，截图保存至 `e:/datacenter2/test-results/` 目录，文件名格式为 `链路编号_时间戳.png`。
5. **超时兜底**：任何单个页面操作超过 15s 未响应，立即标记为 `TIMEOUT` 并跳转至下一链路，不要阻塞整体流程。

## ⚠️ 一、 测试前须知：真假数据（Mock vs Real API）边界声明

为了防止测试人员在测试时对预期的"模拟占位数据"报 Bug，请先了解当前系统中哪些是真实数据库数据，哪些是用于演示交互的挡板数据：

### 🟢 1. 完全真实的后端数据库闭环（涉及 DB 表真实读写）
*   **【系统管理】**：用户管理、角色分配、部门架构、数据范围权限隔离。
*   **【指标知识库】 (`/monitor/lib`)**：所有指标的创建、编辑、删除（CRUD），完全操作了后端的 `sm_indicator_lib` 表。
*   **【复合计算引擎】**：在指标库中点击复合指标的**"试算"**，会真实调用后端的 AST 语法树解析引擎。
*   **【操作审计日志】**：利用后端的 `@Log` 切面，每次核心操作都会在后端生成真实的 `SysOperLog` 记录。

### 🟡 2. 混合型图表（真实请求 + 后端挡板数据）
*   **【全景指标监控】 (`/monitor/index`)**：核心数值基于后台真实逻辑计算。
*   **【钻取分析详情页 (Drawer)】**：历史趋势等接口硬编码了挡板数据，此为演示需要，属正常预期行为。

### 🟣 3. 纯前端 Mock 的演示级模块
*   **【驾驶舱系列】（院长与科室驾驶舱）**：被前端 `mock/cockpit.ts` 拦截。交互验收重点是体验与联动，不涉及深层业务数据。
*   **【门户 Mock】**：被前端 `mock/portal.ts` 拦截，搜索与卡片数据为前端模拟。
*   **【院长驾驶舱 Mock】**：被前端 `mock/president.ts` 拦截。

### 🔗 链路一：指标库导入模板生成与解析（重点验收 CSV 错位与样式）
1.  **路径**：登录系统 > 进入"运营中心" > "指标知识库" (`/monitor/lib`)。
2.  **断言 1 (模版下载)**：下载指标模版，断言表头和示例数据中即使存在括号、逗号等符号，表格列严格对齐，没有任何溢出错位现象。
3.  **断言 2 (上传解析)**：拖拽解析 CSV 文件后，页面列表应自动发生拉取。
4.  **断言 3 (CRUD 完整性)**：新增一条基础指标 → 编辑修改名称 → 确认列表刷新 → 删除该指标 → 确认消失。整个 CRUD 闭环无报错。

### 🧮 链路二：复合图谱计算闭环体验
1.  **路径**：停留在"指标知识库"列表页。
2.  **断言 1 (公式试算)**：找寻任意带有黄色 **"C (复合)"** 标识的指标并点击"试算"，系统将连接真实后端进行解析并弹出数值结果。
3.  **断言 2 (构成树下钻)**：进入**"全院指标监控"**，点击复合指标。断言"指标构成分析"树状图成功分裂，清晰展示出依赖子指标状态。
4.  **断言 3 (指标图谱)**：进入"指标图谱" (`/monitor/graph`)，验证力导向图正常加载且节点可拖拽。

### 📊 链路三：监控看板图全维度交互排雷
1.  **路径**：进入"全院指标监控" (`/monitor`)。
2.  **操作 (分类修正检查)**：断言表格的"分类"列不能留白，应正确渲染为"医疗质量"、"运营效率"等中文文案。
3.  **断言 (下钻无死角)**：点击任一卡片。侧边栏展开后，断言四大区域全部渲染成功，没有任何区域报 `404 No static resource`。
4.  **断言 (导出转义)**：执行"导出数据"，验证带逗号的公式和带有百分号的数据没有引起后续字段的错行。
5.  **断言 (深度分析跳转)**：点击"深度分析"链接，页面跳转至 `/monitor/analysis/:id`，图表与文字均正常渲染。

### 🎡 链路四：驾驶舱多模态极速联动体验
1.  **路径**：进入"决策驾驶舱" (`/cockpit`) > "科室驾驶舱" (`/cockpit/dept`)。
2.  **断言 (沉浸体验与动画复选)**：在下拉框切换科室。断言数字变化跳动特效、医师重置流、图表强制重播放动画必须符合极致流畅预期。
3.  **断言 (院长驾驶舱)**：返回院长驾驶舱 (`/cockpit`)，验证全院级别的趋势图、饼图、排行榜均有数据加载且无白屏。

### 🔍 链路五：门户智能聚合检索验证
1.  **路径**：退回到首页 **"运营决策门户"** (`/portal`)。
2.  **断言 (搜索联想)**：在搜索框中输入关键字。断言探索场景一览卡片做出对应联想反馈，并能降落进而不会丢失 Header 面包屑。
3.  **断言 (卡片跳转)**：点击任一场景卡片，验证跳转路径正确，目标页面渲染完整。

### 🔒 链路六：系统角色与数据权限隔离 (Role & Data Scope)
1.  **路径**：分别使用 `admin` (超管)、`president` (院长)、`director_li` (科室主任)、`wangwu` (护士长) 的凭据进行登录串行验证。
2.  **断言 (菜单隔离)**：非超管角色登录时，不应该出现"系统管理"相关的高敏感配置菜单。
3.  **断言 (数据隔离)**：科室主任只能看见本部门相关数据；超管能穿越部门架构俯瞰全局。
4.  **断言 (敏感指标遮蔽)**：无 `indicatorSensitive` 权限的角色查看敏感指标时，应返回 0 或脱敏文案而非实际值。

### 👁️ 链路七：敏感操作审计与追踪日志 (Audit Logs)
1.  **路径**：以 `admin` 账号登录 > "系统管理" > "日志管理" (`/system/log`)。
2.  **配置核对**：此前如果在指标库中执行了新增/修改指标、或下载了 CSV，对应的切面必须异步存盘。
3.  **断言**：断言审计表格内存在业务模块名为"字典类型"、"角色管理"等字段的请求追踪。点击"详情"必须能渲染出带有参数解析的日志抽屉，无长文本溢出前端。

### ⚙️ 链路八：基础配置数据联动可用性 (Dictionaries)
1.  **路径**：以 `admin` 账号登录 > "系统管理" > "字典管理" (`/system/dict`)。
2.  **断言**：找出一组核心的下拉配置字典（如 "指标频次类型"），执行一次字典数据的状态冻结/解冻操作。
3.  **联动断言**：切换到任意使用该下拉框的业务页面（知识库新增），核验刚才被冻结的字典项是否完美地在 UI 交互表单中被禁用。

## 🔐 五、登录与会话容错验证 (Login & Session)

### 5.1 正常登录流程
1. 访问 `/login` 页面，输入正确凭据，点击登录。
2. **断言**：自动跳转至 `/monitor`（默认首页），左侧菜单正常渲染。

### 5.2 错误凭据容错
1. 输入错误密码或不存在的用户名。
2. **断言**：页面显示友好错误提示（如 "用户名或密码错误"），不出现技术堆栈信息，不跳转。

### 5.3 未登录路由拦截
1. 在未登录状态下，直接访问 `/monitor` 或 `/system/user` 等需要鉴权的页面。
2. **断言**：自动重定向回 `/login`，不出现白屏或 403 裸页面。

### 5.4 Token 过期行为
1. 手动清除 `localStorage` 中的 token，然后刷新当前页面。
2. **断言**：系统应优雅地跳转回登录页，不出现无限循环或空白页。
