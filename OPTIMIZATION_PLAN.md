# 🚀 CloudTech 项目优化与完善方案 (OPTIMIZATION_PLAN)

本方案旨在将 CloudTech 官网从“静态/Mock 驱动”转型为“全动态/内容管理驱动”的现代化网站。

## 📅 阶段一：数据动态化 (消除 Mock)
**目标**：将所有页面上的硬编码内容转移到数据库中。

### 1.1 "关于我们" (About) 页面动态化
- [ ] 后端增加 `timeline` (发展历程) 和 `honors` (荣誉资质) 的数据库表和 API。
- [ ] 前端对接 API，替换 `About.vue` 中的静态数组。

### 1.2 成功案例 (Cases) 模块化
- [ ] 增加 `cases` 数据库表，支持上传案例图片、公司名、统计数据。
- [ ] 在 `Business.vue` 中调用新 API 渲染成功案例。

### 1.3 首页 (Home) 实力数据对齐
- [ ] 增加“站点统计属性”配置，支持动态修改首页显示的“18+年”、“50万+客户”等关键数字。

## 🎨 阶段二：交互与体验升级 (UX)
**目标**：提升站内搜索能力和管理端的易用性。

### 2.1 全站搜索功能
- [ ] 在 Header 增加搜索入口。
- [ ] 实现针对新闻标题、内容及产品名称的模糊搜索功能。

### 2.2 404 及错误处理
- [ ] 创建美观的 `404 Not Found` 页面。
- [ ] 为所有 API 请求添加过渡动画 (Progress bar) 和错误捕获 (Toast 提示)。

### 2.3 后台管理增强
- [ ] 增加“站点配置”模块，允许修改站点 Title、Logo、备案号及底部联系方式。
- [ ] 增加简单的仪表盘统计图表（例如展示新闻点击量或表单提交数）。

## ⚙️ 阶段三：性能与 SEO (Performance)
**目标**：优化加载速度并提升搜索排名。

### 3.1 图像优化系统
- [ ] 引入图片懒加载 (v-lazy) 指令。
- [ ] 替换外链图片为压缩后的本地资源或 CDN 加速链接。

### 3.2 TDK (Title/Description/Keywords) 动态管理
- [ ] 根据后台配置动态生成每个页面的 SEO 标签。
- [ ] 为新闻详情页生成独立的 Meta 信息，方便社交媒体分享。
